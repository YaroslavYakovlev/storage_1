ЗАДАНИЕ К ЛАБОРАТОРНОЙ РАБОТЕ №4 
«Ведение совместной разработки с использованием системы контроля версий MERCURIAL»

Цель работы: ознакомиться с основными возможностями распределённой системы контроля версий mercurial, применять полученные знания для совместной разработки.

Порядок выполнения
1.	Изучить методику ведения совместной разработки с использованием системы контроля версий mercurial.
2.	Скачать и установить ПО для работы с системой контроля версий mercurial.
3.	Открыть графическую оболочку «TortoiseHg Workbench» для работы с системой контроля версий mercurial.
4.	Создать текстовый документ (например, primer.txt). Скопировать в этот файл не менее одной страницы текста домашней работы по данной дисциплине.
5.	Зафиксировать текущую версию созданного в п.4 текстового документа с названием ревизии «Пилотный вариант», предварительно создав хранилище (далее по тексту хранилище 1) в каталоге test рабочего диска ПК.
6.	Создать клон хранилища 1 (далее по тексту хранилище 2) для организации редактирования текстового документа в двух хранилищах с целью проверки возможности одновременной работы с документом более чем одним разработчиком.
7.	Внести изменения в текстовый документ из хранилища 1. Зафиксировать введённые изменения.
8.	Внести изменения в текстовый документ из хранилища 2 (отличные от изменений по п.7 - в другую часть текста). Зафиксировать введённые изменения.
9.	Затянуть изменения их хранилища 2 в хранилище 1, предварительно заполнив параметры хранилища 2 (с которого будут затягиваться изменения) с помощью кнопки «Синхронизация».
10.	В хранилище 1 провести слияние версий текстовых документов. Проверить корректность работы программы после слияния версий (в объединённой версии – в текстовом документе из хранилища 1 должны присутствовать текстовые правки п.7 и 8).
10. Отчёт должен содержать: указание цели работы, краткую последовательность её выполнения, распечатку исходного текста, а также текстов с правками по п.7,8,10, распечатку перечня ревизий хранилища 1 и 2 из рабочего окна графической оболочки «TortoiseHg Workbench», выводы по проделанной работе.
